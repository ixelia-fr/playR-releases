Installer & configurer playR

Tout est expliqué pas à pas — même si vous débutez.

Le principe

playR = un serveur (qui héberge la bibliothèque) + des clients (les PC qui s'y connectent).

  1. Le serveur tourne sur un PC, un CasaOS, un NAS ou un Proxmox.
  2. Le client (app Windows) se connecte à l'adresse du serveur (ex. 192.168.1.10:3000).
  3. Vous connectez vos comptes (Steam, Epic…) et synchronisez.

Vous avez besoin d'un seul serveur sur votre réseau ; tous vos PC clients pointent dessus.

Serveur — sur un PC Windows

Le plus simple : au moment d'installer playR (télécharger l'installeur), cochez « Serveur » en plus du client.

  • Une petite fenêtre playR Serveur s'ouvre et affiche l'adresse à donner aux autres PC.
  • Fermer la fenêtre la réduit dans la barre système (près de l'horloge) — le serveur continue de tourner.
  • Pour l'arrêter : clic droit sur l'icône → Arrêter.

Serveur — sur CasaOS

  1. App Store → Installer une app personnalisée.
  2. Cliquez sur l'icône Importer, en haut à droite de la fenêtre.
  3. Copiez le contenu de ce fichier, collez-le, puis Envoyer.
  4. Cliquez sur Installer.
  5. Dans la fenêtre qui s'ouvre : Installer, puis Ouvrir.

Adresse à donner aux clients : http://[IP-du-CasaOS]:3000.

Serveur — sur Synology / NAS

Avant tout : votre Synology supporte-t-il Docker ?
Docker n'est disponible que sur les Synology compatibles (surtout modèles Intel/x86 — séries « + » et milieu/haut de gamme). Sur beaucoup de modèles ARM / entrée de gamme, l'app n'apparaît pas du tout dans le Centre de paquets, quelle que soit la version de DSM. Dans ce cas, hébergez le serveur ailleurs : un PC avec l'app Windows « Serveur », un mini-PC, un Raspberry Pi…
Pour vérifier : liste des modèles compatibles Container Manager.

Cas des Synology ARM (ex. DS218 — puce RTD1296) : Synology ne propose PAS Docker / Container Manager pour ces modèles dans le Centre de paquets. Une solution communautaire existe (script 007revad « ContainerManager for all armv8 », DSM 7.2+) qui installe Container Manager sur ces modèles — non officiel, à vos risques.
⚠️ De plus, l'image playR est pour l'instant en amd64 uniquement ; le support arm64 (NAS ARM, Raspberry Pi) est en cours. En attendant, sur ces machines, le plus simple est d'héberger le serveur ailleurs (mini-PC, CasaOS, ou l'app Windows « Serveur »).

Le nom de l'app dépend de votre version de DSM :

  • DSM 7.2 / 7.3 → « Container Manager »
  • DSM 7.0 / 7.1 → « Docker »

Si l'app est disponible dans le Centre de paquets, installez-la, ouvrez-la, puis :

  1. Projet → Créer.
  2. Nom : playr · chemin : un dossier (ex. /docker/playr).
  3. Source : Créer docker-compose.yml, et collez :
services:
  playr:
    image: ghcr.io/ixelia-fr/playr:latest
    container_name: playr
    restart: unless-stopped
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATA_PATH=/app/data
    volumes:
      - /volume1/docker/playr/data:/app/data
  1. Suivant → Terminer. Container Manager télécharge l'image et démarre.
  2. Adresse pour les clients : http://[IP-du-Synology]:3000.
Architecture : l'image est en amd64 (Synology Intel : OK). Sur un Synology ARM, il faut une image multi-arch — demandez et on l'ajoute.

Serveur — sur Proxmox

Deux options :

  • Un conteneur LXC avec Docker installé, puis la commande Docker générique.
  • Une petite VM (Debian/Ubuntu) avec Docker, idem.

Serveur — Docker (la méthode universelle)

Le serveur est une image Docker : elle tourne sur n'importe quel hôte Docker (un simple serveur Linux, un mini-PC, un Raspberry Pi Intel/amd64…). C'est la base de tous les autres cas.

mkdir -p ./playr/data
docker run -d --name playr --restart unless-stopped \
  -p 3000:3000 \
  -e DATA_PATH=/app/data \
  -v $(pwd)/playr/data:/app/data \
  ghcr.io/ixelia-fr/playr:latest

Ou avec un docker-compose.yml puis docker compose up -d.

Serveur — Portainer, Unraid, TrueNAS, OMV…

Toutes ces plateformes exécutent Docker → c'est le même fichier compose partout. Repères par outil :

  • Portainer / Dockge : Stacks → Add stack → collez le compose → Deploy.
  • Unraid : plugin Compose Manager (ou un template Docker) — image ghcr.io/ixelia-fr/playr:latest, port 3000, volume vers /app/data.
  • TrueNAS SCALE : Apps → Custom App / Docker Compose — même image, même port, même volume.
  • OpenMediaVault : plugin compose (omv-extras) → collez le compose.

Dans tous les cas : image ghcr.io/ixelia-fr/playr:latest, port 3000, volume persistant sur /app/data. Adresse pour les clients : http://[IP-de-la-machine]:3000.

Mettre à jour le serveur

Automatique (inclus par défaut). Le docker-compose.yml fourni embarque un service playr-updater qui met playR à jour tout seul chaque nuit. Rien à configurer, rien à taper. Il ne touche qu'au conteneur playR.

Manuellement (repli). Une notification (🔔 en haut à droite) vous prévient quand une version existe. Si vous préférez déclencher vous-même, ou si vous avez installé l'image seule (sans le compose), la notification affiche la commande à lancer sur la machine hôte :

docker compose pull && docker compose up -d

Serveur Windows : il se met à jour tout seul avec l'application.

Installer le client

Téléchargez l'installeur, cochez Client. Au premier lancement, saisissez l'adresse de votre serveur (ex. 192.168.1.10:3000). Vous pouvez en changer à tout moment (menu ⚙ → Changer de serveur).

Steam

Deux façons, au choix :

  • Steam Connector (recommandé — inclut les jeux de la famille) : ⚙ Paramètres → Steam → entrez votre identifiant et mot de passe Steam, validez le code Steam Guard. Connexion unique (un jeton est stocké, rien n'est renvoyé ailleurs).
  • Ou SteamID + clé API (jeux possédés uniquement, sans la famille) : votre SteamID64 (17 chiffres, ex. 7656… — trouvable via steamid.io) et une clé gratuite sur steamcommunity.com/dev/apikey.

Epic Games

  1. ⚙ Paramètres → Epic → bouton « Ouvrir Epic (obtenir le code) ».
  2. Connectez-vous à Epic si besoin : une page JSON s'affiche dans le navigateur.
  3. Copiez la valeur du champ authorizationCode (une longue suite de caractères).
  4. Collez-la dans l'app et cliquez Valider. Connexion unique (le refresh token est conservé).

GOG

GOG n'a pas d'API de connexion : on lit votre profil public.

  1. Sur gog.com, rendez votre profil public (Paramètres du profil → Confidentialité).
  2. Ouvrez l'onglet JEUX de votre profil et copiez l'URL, du type https://www.gog.com/u/PSEUDO/games.
  3. Collez-la dans ⚙ Paramètres → GOG → Synchroniser.

Amazon Games

Amazon ne donne pas de jeton simple : un petit script récupère votre bibliothèque depuis votre PC (l'appli Amazon Games doit être installée et connectée).

  1. ⚙ Paramètres → Amazon → « Télécharger le script PowerShell ».
  2. Sur votre PC Windows : clic droit sur le script → Exécuter avec PowerShell.
  3. Il crée un fichier amazon_games.json et ouvre son dossier.
  4. Glissez ce fichier dans la zone d'import de l'app (⚙ Paramètres → Amazon).

Obtenir sa clé IGDB (jaquettes & métadonnées)

IGDB passe par un compte Twitch développeur (gratuit) :

  1. Allez sur dev.twitch.tv/console → Register Your Application.
  2. Nom au choix · OAuth Redirect URL : http://localhost · Catégorie : Application Integration.
  3. Récupérez le Client ID et générez un Client Secret.
  4. Dans playR → ⚙ Paramètres → IGDB : collez Client ID + Client Secret.

Puis lancez la synchronisation des métadonnées : jaquettes, descriptions et notes se remplissent.
